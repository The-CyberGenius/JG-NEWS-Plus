Architectural Highlights

⚡ Direct-to-CDN Uploads

Browser uploads PDFs directly to Cloudinary, bypassing the 4 MB Vercel serverless body limit and removing backend bandwidth as a bottleneck.

📡 Stateless Serverless API

Express app exported as a Vercel handler — auto-scaling, zero idle cost, sub-100ms cold starts.

🧠 Smart Pagination

Article list endpoint supports field projection (?fields=summary), pagination, and stale-while-revalidate caching to keep payloads small.

🔁 Optimistic Frontend

React Context with ref-guarded fetch deduplication; admin actions trigger targeted refetch instead of full reload.

07 Feature Breakdown (Cont.)

🤖 AI News Sync (MCP) — Detailed Subsystem

One of the most technically involved subsystems. The AI News Sync module aggregates content from multiple Indian news sources, decodes Google News redirect URLs, extracts full article content, and stages it for editor review before publishing.

  • Multi-source RSS: NDTV Hindi, Amar Ujala, Google News (Rajasthan / India / World feeds), and additional configurable sources.
  • Google News URL Decoder: Custom integration to resolve obfuscated URLs to original publisher links.
  • Article Extractor: Pulls full HTML content, og:image fallbacks, and og:description metadata automatically.
  • Review Modal: Editor sees title, image, excerpt, and full content side-by-side before clicking "Publish" — protects editorial quality.
  • Categorized Sync: Separate one-click syncs for Rajasthan, India, and World categories.
  • Bulk Cleanup Tooling: Backend script to remove all AI-synced articles with backup-to-JSON safety net.

📰 E-Newspaper System — Detailed Subsystem

  • Direct Cloudinary Upload: Bypasses Vercel's strict 4MB serverless body limit by uploading from the browser directly to Cloudinary using unsigned upload presets.
  • Custom react-pdf Reader: Built-in viewer with zoom controls, fullscreen mode, keyboard shortcuts (←/→/Esc), and responsive page width calculation.
  • Hindi cMap Support: Bundled cMap files ensure proper Devanagari rendering even for complex ligatures inside PDFs.
  • Auto-Edition Numbering: Admin form auto-increments edition number based on previous publications.
  • Edition Catalog: Public archive with sortable list of all past editions.

📺 Live TV & Weather — Detailed Subsystem

  • YouTube Live Integration: Admin pastes a YouTube embed URL; site automatically displays it on the Live page and the Home banner.
  • Auto-Play Configuration: Live page autoplays the stream on mount without user interaction (browser policy dependent).
  • Live Badge: Animated "LIVE" pulse indicator injected throughout the site navigation.
  • Batched Weather API: Real-time temperature pulled from Open-Meteo API in a single batched request to prevent rate limiting. Covers 16 key cities.

09 Performance & Optimization

Performance was treated as a first-class feature, not an afterthought. The following dedicated work was carried out to ensure the platform feels instantaneous on both desktop and mobile networks.

🎯 Optimization Pass Highlights

Technique Impact
API Pagination & Projection Public site loads only 30 articles in summary projection (no heavy content field) — ~70% smaller payload on initial visit.
Mongoose .lean() Queries Skips Mongoose document instantiation overhead — database reads return faster with less memory usage.
Gzip Compression Middleware All API responses compressed; typical 60–80% size reduction over the wire.
Cache-Control Headers public, max-age=30, stale-while-revalidate=60 — instant cached responses with background revalidation.
Fetch Deduplication React ref-guarded context prevents duplicate fetches when components rapidly mount/unmount.
Parallel API Calls Promise.all for articles + categories + settings — total wait time = max instead of sum.
Lazy Route Code-Splitting All pages and admin modules loaded via React.lazy() — initial bundle stays small.
Image loading="lazy" Off-screen images deferred until scrolled into view; reduces initial network spike.
CSS Marquee (GPU-accelerated) Ticker uses transform: translateX with will-change — runs on the compositor, no JS frame loop.
Skeleton Loaders Perceived performance boost — users see structure immediately, never blank screens.
