02Executive Summary

JG News Plus is a production-ready, full-stack Hindi news portal & digital E-Newspaper platform engineered for the Rajasthan readership. The platform combines a high-performance React frontend with a Node.js / MongoDB backend, integrated AI-powered news syncing, professional admin tooling, and direct cloud media delivery.

This document details two full months of dedicated engineering effort spanning 81 commits, 500,000+ lines of code touched, and the integration of 15+ third-party services. The system is currently live in production at jgnews.live serving daily traffic.

03Project Overview & Scope

The brief required a platform that could be operated by non-technical journalists while delivering enterprise-grade performance, SEO, and visual polish.

Architectural Highlights

⚡ Direct-to-CDN Uploads

Browser uploads PDFs directly to Cloudinary, bypassing the 4 MB Vercel serverless body limit and removing backend bandwidth as a bottleneck.

📡 Stateless Serverless API

Express app exported as a Vercel handler — auto-scaling, zero idle cost, sub-100ms cold starts.

🧠 Smart Pagination

Article list endpoint supports field projection (?fields=summary), pagination, and stale-while-revalidate caching to keep payloads small.

🔁 Optimistic Frontend

React Context with ref-guarded fetch deduplication; admin actions trigger targeted refetch instead of full reload.

07Feature Breakdown (continued)

🤖 AI News Sync (MCP) — Detailed

One of the most technically involved subsystems. The AI News Sync module aggregates content from multiple Indian news sources, decodes Google News redirect URLs, extracts full article content, and stages it for editor review before publishing.

  • Multi-source RSS: NDTV Hindi, Amar Ujala, Google News (Rajasthan / India / World feeds), and additional configurable sources.
  • Google News URL Decoder: Custom integration with google-news-url-decoder to resolve obfuscated Google News URLs to original publisher links.
  • Article Extractor: @extractus/article-extractor pulls full HTML content, og:image fallbacks, and og:description metadata.
  • Review Modal: Editor sees title, image, excerpt, and full content side-by-side before clicking "Publish" — protects editorial quality.
  • Categorized Sync: Separate one-click syncs for Rajasthan, India, and World categories.
  • Bulk Cleanup Tooling: Backend script to remove all AI-synced articles with backup-to-JSON safety net.

📰 E-Newspaper System — Detailed

  • Direct Cloudinary Upload: Bypasses Vercel's 4 MB serverless body limit by uploading from browser → Cloudinary directly using unsigned upload presets.
  • Custom react-pdf Reader: Built-in viewer with zoom controls, fullscreen mode, keyboard shortcuts (←/→/Esc), responsive page width.
  • Hindi cMap Support: Bundled cMap files ensure proper Devanagari rendering even for complex ligatures.
  • Auto-Edition Numbering: Admin form auto-increments edition number based on previous publications.
  • Edition Catalog: Public archive with sortable list of all past editions.

📺 Live TV & Media — Detailed

  • YouTube Live Integration: Admin pastes a YouTube embed URL; site shows it on Live page and Home banner.
  • Auto-Play Configuration: Live page autoplays the stream on mount.
  • Live Badge: Animated "LIVE" pulse indicator throughout site.
  • Inline Video in Articles: Articles can include a videoUrl field that renders an embedded video alongside text.

🌡️ Live Weather — Detailed

  • Real-time temperature pulled from Open-Meteo API in a single batched request.
  • 16 cities covered: Ratangarh, Jaipur, Jodhpur, Udaipur, Kota, Bikaner, Ajmer, Alwar, Bhilwara, Sikar, Pali, Mt. Abu, Jaisalmer, Chittorgarh, Bharatpur, Ganganagar.
  • Weather-code → emoji mapping for visual icons (☀️ ⛅ 🌧️ ❄️ ⛈️).
  • Horizontal scrollable strip on home page with brand-colored cards.

09Performance & Optimization Work

Performance was treated as a first-class feature, not an afterthought. The following dedicated work was carried out to ensure the platform feels instantaneous on both desktop and mobile.

🎯 Optimization Pass Highlights

Technique Impact
API Pagination & Field Projection Public site loads only 30 articles in summary projection (no heavy content field) — ~70% smaller payload on initial visit.
Mongoose .lean() Queries Skips Mongoose document instantiation overhead — reads return faster, less memory.
Gzip Compression Middleware All API responses compressed; typical 60–80% size reduction over the wire.
Cache-Control Headers public, max-age=30, stale-while-revalidate=60 — instant cached responses with background revalidation.
Fetch Deduplication React ref-guarded context prevents duplicate fetches when components remount.
Parallel API Calls Promise.all for articles + categories + settings — total wait time = max instead of sum.
Lazy Route Code-Splitting All pages and admin modules loaded via React.lazy() — initial bundle stays small (~136 KB gzipped).
Image loading="lazy" Off-screen images deferred until scrolled into view; reduces initial network spike.
Cloudinary CDN Delivery Media served from global edge nodes, not from app server.
Dedicated Article Endpoint Article reader fetches full content directly via /api/articles/:id — full content not bundled in homepage payload.
CSS Marquee (GPU-accelerated) Ticker uses transform: translateX with will-change — runs on the compositor, no JS frame loop.
Skeleton Loaders Perceived performance boost — users see structure immediately, never blank screens.
